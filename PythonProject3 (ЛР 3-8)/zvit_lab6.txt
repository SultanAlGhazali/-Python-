ВІДПОВІДІ НА КОНТРОЛЬНІ ПИТАННЯ (ВЕБ-ПАРСИНГ ТА HTTP-ЗАПИТИ):
--------------------------------------------------------------------------------------
1. У чому полягає різниця між HTTP-запитами типу GET та POST?
Запит GET використовується виключно для запиту та отримання інформації з сервера
(при цьому параметри передаються відкрито безпосередньо у рядку URL). Запит POST
застосовується для надсилання даних на сервер (наприклад, при авторизації або
заповненні форм), а самі дані передаються приховано всередині тіла запиту.

2. Яку роль відіграє бібліотека "requests" під час створення веб-парсерів?
Бібліотека 'requests' відповідає за мережеву взаємодію з веб-сервером. Вона відправляє
HTTP-запити на вказану URL-адресу, отримує відповідь від сервера, обробляє статус-коди
(наприклад, 200 OK) і завантажує вихідний код сторінки (HTML-код) у вигляді тексту.

3. Яке призначення бібліотеки "BeautifulSoup" (bs4)?
BeautifulSoup призначена для розбору (парсингу) сирого HTML або XML коду сторінки.
Вона перетворює текст сторінки на структуроване дерево об'єктів і надає розробнику
зручні інструменти для пошуку конкретних елементів за тегами, класами чи ідентифікаторами
(методи .find() та .find_all()).

4. Навіщо у HTTP-запитах передавати заголовок "User-Agent"?
Заголовок 'User-Agent' повідомляє серверу інформацію про те, який саме браузер та операційна
система роблять запит. Веб-парсери передають цей заголовок, щоб імітувати дії реальної людини
через браузер і уникати автоматичного блокування (бан-систем та капч) з боку захисту сайту.

5. Що таке статус-коди HTTP і які з них є найбільш поширеними?
Статус-коди — це тризначні числа, які сервер повертає у відповідь на запит клієнта для
позначення результату операції. Основні коди:
- 200 (OK) — запит успішно виконано;
- 403 (Forbidden) — доступ до сторінки заборонено;
- 404 (Not Found) — сторінку або файл не знайдено на сервері;
- 500 (Internal Server Error) — внутрішня помилка самого сервера.
